Release v1.3.0 - My Issues Tab & Performance Optimization

## 🎯 Major Feature: "My Issues" Tab

### Personal Analytics Dashboard
- **New "My Issues" Tab**: Dedicated section showing only issues assigned to current user
- **User Detection**: Automatically detects logged-in user via Jira API (`/rest/api/2/myself`)
- **Personalized Metrics**:
  - My Total Scans
  - My Issues Found
  - My Avg Issues/Scan
  - My Issues Fixed
- **My Recent Scans**: Project-wise tabs filtered by assignee
- **User Info Display**: Shows current logged-in user with visual confirmation

### Performance Optimizations
- **Efficient Filtering**: Direct array filtering instead of full data reload
- **Assignee Tracking**: Added assignee email and display name to scan data
- **No Additional API Calls**: Uses existing scan data for filtering
- **Instant Load**: My Issues tab loads instantly from cached data

## ✨ Enhancements

### Data Tracking
- **Assignee Field**: Now tracks `assignee` (email) and `assigneeDisplayName` in scan entries
- **Backward Compatible**: Existing scans without assignee still work (shown as "Unassigned")

### UI Improvements
- **Assignee Column**: Added assignee information to timeline view
- **User Status Indicator**: Green checkmark when user detected, red warning if not
- **Responsive Grid**: Timeline grid expanded to accommodate assignee info

### Code Refactoring
- **Reusable Functions**: `displayScansWithTabs()` function for both All Scans and My Issues
- **getCurrentUser()**: Centralized user detection function
- **loadMyIssues()**: Dedicated function for My Issues metrics calculation

## 🐛 Bug Fixes
- None (new feature release)

## 📊 Feature Details

### How It Works
1. User opens Analytics Dashboard
2. Extension fetches current user from Jira API
3. Filters scan data by assignee email/name
4. Displays personalized metrics and scans
5. Project tabs show only user's issues

### Fallback Behavior
- If user detection fails: Shows warning message
- If no scans for user: Shows helpful "No data" message
- Unassigned issues: Labeled as "Unassigned" in timeline
