🌐 Global JCP Settings

Configure your Jira environment and workflow settings.

Jira Hosting

Select your Jira environment type. This determines which validation rules and features are available.

Workflow

Select the workflow your team follows. This affects validations and available features across the extension.

Time Log Restriction

When enabled, the extension will block time logging on Jira issues that have pending validation errors.

Optional Validations

Check to enable these validations

Tempo Timesheet Settings

On Fridays, if logged hours are below target, a reminder will appear. On the last working day of the month, a timesheet submission reminder will show if not submitted.

Mandatory Validations

🔵 Both Workflows  |  🟡 Standard Only (Story → Sub-tasks)  |  🟢 New Workflow Only (Story → Tasks)

Field Requirements

  • 🔵 Description required for Story and Bug
  • 🔵 Assignee required for Story, Task, Sub-task, Bug
  • 🔵 Priority required for Story, Task, Sub-task, Bug
  • 🔵 Financial Category required for Story, Task, Bug, Sub-task
  • 🔵 Story Points required for Story (when status beyond NEW)
  • 🟡 Original Estimate required for Sub-task
  • 🟢 Original Estimate required for Task
  • 🔵 Task must have Epic Link
  • 🔵 Bug must have Epic Link
  • 🔵 Epic must have Parent Link

Time Tracking

  • 🔵 Time logging not allowed on Story or Epic
  • 🔵 Time logged but status still in To Do
  • 🟡 Sub-task 100% logged but still open
  • 🟢 Task 100% logged but still open

Workflow & Structure

  • 🟡 Story beyond NEW must have Sub-tasks
  • 🟡 Story should be closed when all Sub-tasks and linked Bugs are Done
  • 🟢 Story beyond NEW must have linked Tasks ("Is a Child of")
  • 🟢 Story should be closed when all linked Tasks are Done
  • 🔵 Story/Task/Bug in progress must be assigned to a Sprint (except Blocked)
  • 🟢 Task/Bug in progress must be assigned to a Sprint (except Blocked)
  • 🟢 Story must NOT be assigned to a Sprint (Tasks handle sprint assignment)

Version & Release

  • 🔵 Fix Version marked as Released requires status Done
  • 🔵 Fix Version with past release date must be marked as Released

Target Dates

  • 🟡 Target Start overdue for Story/Sub-task in To Do
  • 🟡 Target End overdue for Story/Sub-task not completed
  • 🟢 Target Start overdue for Story/Task in To Do
  • 🟢 Target End overdue for Story/Task not completed
